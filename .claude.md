## Excalidraw diagram generation

### Pipeline
1. Python generator script (`talks/gen_*.py`) ŌåÆ `.excalidraw` JSON
2. `excalidraw-to-svg` ŌåÆ `.svg` (hand-drawn style)
3. `talks/fix_excalidraw_svg.py` ŌåÆ fix `y="NaN"` text bug

### Rendering
- Tool: `excalidraw-to-svg@3.1.0` (in `~/.npm/_npx/0437aa2f6562dfa6/`)
- Must `cd` into the npx cache dir to run: `cd ~/.npm/_npx/0437aa2f6562dfa6 && npx excalidraw-to-svg /path/to/file.excalidraw`
- Requires `canvas` npm package (needs system libs: `cairo-devel pango-devel libjpeg-turbo-devel giflib-devel librsvg2-devel` on Fedora)

### Critical: y="NaN" text bug
- `excalidraw-to-svg` produces `y="NaN"` for ALL text elements (known bug in `@excalidraw/utils`)
- Post-process with `talks/fix_excalidraw_svg.py` to fix
- Fix formula: `y = halfH + fontSize * 0.35` where halfH comes from the `rotate(0 halfW halfH)` transform
- For multiline text groups, distribute lines evenly around center

### Generator conventions
- Use `roughness=2, fontFamily=3` (Cascadia Code), transparent background
- **NEVER use bound text** (`containerId`/`boundElements`). Always use standalone text with `containerId: None`. Bound text causes rendering failures.
- Position text manually inside shapes by calculating coordinates
- `roundness: {"type": 3}` for rects, `{"type": 2}` for lines
- For arrows: use `type: "line"` with `endArrowhead: "arrow"`

### Embedding images (logos etc.)
- Excalidraw supports embedded images via `files` dict with base64-encoded PNG data
- Use `type: "image"` elements with `fileId` referencing keys in `files`
- File ID: SHA1 hash of PNG binary data
- Convert SVG logos to PNG via `cairosvg` Python package: `cairosvg.svg2png(url=path, write_to=out, output_height=30)`
- Watch out for white-on-transparent logos ŌĆö recolor SVG fills before converting

### Critical: excalidraw-to-svg only takes ONE file at a time
- The CLI uses `argv[2]` as input, `argv[3]` as output path
- Passing multiple .excalidraw files will corrupt the second file (SVG written over it)
- Always render one file per invocation

### Slide renumbering
Use this Python one-liner after reordering slides:
```
python3 -c "
import re
with open('talks/picodata-renegade-underdogs.markdown') as f:
    lines = f.readlines()
n = 0
for i, line in enumerate(lines):
    if re.match(r'^## \d+\. ', line):
        n += 1
        lines[i] = re.sub(r'^## \d+\. ', f'## {n}. ', line)
with open('talks/picodata-renegade-underdogs.markdown', 'w') as f:
    f.writelines(lines)
print(f'Renumbered {n} slides')
"
```
